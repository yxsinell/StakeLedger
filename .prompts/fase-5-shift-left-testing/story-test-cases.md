## ¿Por qué el cambio?

El nombre "Acceptance Test Plan" refleja mejor la conexión entre:

- **Acceptance Criteria (AC)** → Define CUÁNDO la story está lista
- **Acceptance Test Cases** → Define CÓMO validar cada AC

Esta alineación hace más clara la trazabilidad:

```
User Story
    └── Acceptance Criteria (AC)
            └── Acceptance Test Cases (en acceptance-test-plan.md)
```
